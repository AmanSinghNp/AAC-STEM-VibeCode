# Student Cheatsheet

## Your Mission

Make a small website and publish it on the internet.

You are allowed to use AI, but you are still the director. Your job is to describe what you want, test what comes back, and improve it.

## The 5-Step Loop

1. Ask for a small change.
2. Read what changed.
3. Preview the site.
4. Fix anything weird.
5. Save and deploy.

## Edit These First

Open `site.config.js`.

Change:

- `name`
- `tagline`
- `description`
- `theme`
- `features`
- `gallery`
- `links`

## Good Prompts

```text
Make the website feel like a gaming team site. Keep it easy to edit.
```

```text
Add a new section for upcoming events. Use the existing style.
```

```text
Make the cards more colourful, but keep the text readable on mobile.
```

```text
Explain this file like I am new to coding.
```

```text
Find the bug. The button text changed but the link does not work.
```

## Words You Need

- **HTML:** the page content
- **CSS:** the styling
- **JavaScript:** the interactivity
- **GitHub:** where the code lives
- **Vercel:** where the website goes live
- **Deploy:** publish the latest version
- **Repo:** a project folder on GitHub

## Do Not Paste

Never paste passwords, tokens, private messages, or personal details into AI or a public website.
